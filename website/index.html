Satra crypto wallet.

A quiet Android wallet for people who want direct control of their keys, assets, networks, and activity.

Create a wallet, import an existing one, manage multiple local wallets, receive and send supported assets, review activity, track prices in local currency, and keep custody on your device. No accounts. No ads. No analytics.

Designed around ownership.

Satra is built as wallet software, not an exchange. It keeps the important decisions visible: the asset, the network, the address, the amount, the fee, and the backup that controls the wallet.

Local-first self-custody

Recovery phrases, private keys, passcodes, and biometric settings remain on your device. Satra does not create a cloud account and cannot recover your wallet for you.

Multiple wallet types

Create a new wallet, restore from a recovery phrase, import chain-specific private keys, or add watch-only addresses for monitoring public activity without signing.

Network-aware send and receive

Asset and network selection are separated so USDT, USDC, ETH, Bitcoin-family coins, and other supported assets are never treated as one-size-fits-all transfers.

Activity with context

Transaction history is grouped with direction, amount, network, token logo, timestamp, status, and explorer context where supported by the public providers.

Local currency values

Market prices are cached in USD first, then converted to the selected local currency so the whole interface can update quickly when the currency changes.

Security that stays understandable

Use app passcode, optional biometric unlock, auto-lock, wallet reset protection, private-key backup screens, and explicit warnings before sensitive actions.

Every screen keeps the chain in view.

Sending and receiving crypto is not just an address field. Satra keeps the selected asset, selected network, amount, fee asset, and recipient context visible before the final review.

Privacy position

No accounts, no ads, no analytics SDKs.

Satra does not ask for email, phone number, username, or a hosted account. Public wallet data may still be sent to public blockchain infrastructure so the app can sync balances, estimate fees, and broadcast transactions.

What stays private

Keys are not a server feature.

Recovery phrases and private keys stay on-device. The backup screen is treated as sensitive because anyone with those secrets can control the wallet.

What becomes public

Blockchains are public systems.

Signed transactions, wallet addresses, balances, hashes, and network identifiers can be visible to public RPC, API, Electrum, explorer, indexer, and blockchain providers.

Support without pretending.

Satra supports only the networks and assets listed in the public project documentation. The app is designed to be honest when provider data is delayed, partial, or unavailable.

Bitcoin family, EVM, and selected non-EVM networks.

Supported networks include Bitcoin, Bitcoin Cash, Dogecoin, Litecoin, Ethereum, Arbitrum, Base, Optimism, Scroll, zkSync Era, Polygon, BNB Chain, opBNB, Avalanche, Celo, Kava EVM, Aptos, NEAR, Polkadot, XRP Ledger, Solana, Stellar, Sui, TON, TRON, and Kava.

Open source

Open-source by design.

Satra is built in Kotlin with Jetpack Compose and Material 3. The public repository includes app code, brand assets, supported assets, privacy policy, terms, and localization tooling so reviewers can inspect how the wallet works.
